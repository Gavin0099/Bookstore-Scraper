**專案名稱**：Bookstore Scraper  
**第一目標網站**：三采文化（suncolor.com.tw）  
**文件狀態**：Draft  
**更新日期**：2026-03-05

---

## 1. 目標與範圍

### 1.1 目標
自動抓取三采文化官網的書籍資料，篩選出有效在售品項，輸出結構化 Excel 報表，供內部書單管理或比價使用。

### 1.2 範圍（v1）
- 資料來源：三采文化官網書籍目錄頁
- 抓取欄位：書名、定價、ISBN
- 排除規則：缺書品項（庫存狀態 = 缺書/絕版/停售）
- 輸出格式：`.xlsx`

### 1.3 範圍外（Out of Scope）
- 其他書商（後續版本擴充）
- 折扣價、電子書、有聲書
- 會員登入後的特殊價格
- 即時庫存數量（僅判斷是否缺書）

## 2. 資料規格

### 2.1 抓取欄位定義

| 欄位 | 型別 | 說明 | 必填 |
|------|------|------|------|
| 書名 | String | 完整書名（含副標題） | ✅ |
| 定價 | Integer | 台幣售價，純數字，不含「NT$」符號 | ✅ |
| ISBN | String | 13 碼 ISBN，格式 `978XXXXXXXXXX` | ✅ |
| 庫存狀態 | Enum | 用於過濾，不輸出至 Excel | — |

### 2.2 過濾規則

- **排除**：庫存狀態包含以下任一關鍵字的品項
  - 缺書、缺貨、絕版、停售、預購（可擴充）
- **保留**：可正常加入購物車 / 顯示「立即購買」的品項
- **ISBN 缺失**：若頁面無 ISBN 資訊，該筆記錄**跳過並記 log**，不輸出

### 2.3 資料品質規則

- 定價為 0 或負數 → 跳過並記 log
- ISBN 非 13 碼數字 → 跳過並記 log
- 書名為空 → 跳過並記 log
- 重複 ISBN → 保留第一筆，後續略過並記 log

## 3. 輸出規格

### 3.1 Excel 檔案格式

- 檔名：`suncolor_books_YYYYMMDD_HHMMSS.xlsx`
- 工作表名稱：`書單`
- 欄位順序：`書名` | `定價` | `ISBN`
- 第一列為標題列（粗體）
- 資料從第二列開始

### 3.2 欄位格式

| 欄位 | Excel 格式 |
|------|-----------|
| 書名 | 文字，左對齊 |
| 定價 | 數值，整數，右對齊 |
| ISBN | 文字（避免被轉為科學記號） |

### 3.3 執行 Log

- 輸出同目錄下 `scraper_YYYYMMDD_HHMMSS.log`
- 記錄內容：
  - 執行起訖時間
  - 總抓取頁數
  - 總處理筆數
  - 成功輸出筆數
  - 跳過筆數（含原因分類）
  - 錯誤訊息（HTTP 失敗、解析失敗等）

## 4. 爬蟲行為規格

### 4.1 入口策略

- 從三采官網書籍分類目錄頁開始巡覽
- 支援分頁（pagination）自動翻頁至最後一頁
- 每頁取得書籍列表 → 逐一進入個別書籍詳情頁抓取完整資訊

### 4.2 請求規格

| 項目 | 規格 |
|------|------|
| HTTP 請求間隔 | 1–3 秒隨機（避免被封鎖） |
| User-Agent | 模擬一般瀏覽器 |
| 逾時設定 | 單頁請求逾時 15 秒 |
| 重試次數 | 失敗最多重試 3 次，每次間隔加倍（Exponential Backoff） |
| robots.txt | 遵守（先行檢查並記 log） |

### 4.3 錯誤處理

| 情境 | 處理方式 |
|------|---------|
| HTTP 4xx | 記 log，跳過該頁 |
| HTTP 5xx | 重試 3 次，仍失敗則記 log 跳過 |
| 解析失敗 | 記 log，跳過該筆 |
| 網路逾時 | 重試 3 次 |
| 任何未預期例外 | 捕捉並記 log，繼續下一筆 |

## 5. 技術規格建議

### 5.1 語言與框架

| 層 | 建議選項 |
|----|---------|
| 爬蟲 | Python 3.10+ |
| HTTP | `requests` + `BeautifulSoup4` 或 `httpx` |
| 動態頁面備援 | `playwright`（若目標頁面需 JS 渲染） |
| Excel 輸出 | `openpyxl` |
| Log | `logging`（標準庫） |

### 5.2 專案結構（建議）

```
bookstore-scraper/
├── main.py                  # 入口點
├── scrapers/
│   └── suncolor.py          # 三采爬蟲實作
├── models/
│   └── book.py              # Book dataclass
├── output/
│   └── excel_writer.py      # Excel 輸出邏輯
├── utils/
│   ├── logger.py
│   └── http_client.py
├── config.yaml              # 設定檔（delay、retry 等）
├── requirements.txt
└── README.md
```

## 6. 驗收條件（Acceptance Criteria）

| # | 條件 |
|---|------|
| AC-01 | 執行後產出 `.xlsx`，包含書名、定價、ISBN 三欄 |
| AC-02 | 缺書品項不出現在 Excel 中 |
| AC-03 | ISBN 格式正確（13 碼字串），不被 Excel 轉為科學記號 |
| AC-04 | 執行後產出 `.log`，記錄跳過筆數與原因 |
| AC-05 | 單次執行遇到單頁錯誤時，程式不中止，繼續處理其餘頁 |
| AC-06 | 請求間隔符合設定（1–3 秒），不暴力爬取 |
| AC-07 | 重複 ISBN 僅保留一筆 |
